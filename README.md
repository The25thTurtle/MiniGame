# Simplistic Gaming Interface Design & Learning Experience Study

## Overview
This project explores how different levels of interface complexity affect user learning, performance, and experience in a simple memory matching game.

Users interact with two versions of the same game:
- A **simple interface version**
- A **complex interface version**

The goal is to compare how interface design influences:
- Learning speed
- Accuracy
- Completion time
- User enjoyment
- Perceived difficulty

---

## Project Structure

### 1. Simple Version
- File: `simple_game.html`
- Features:
  - Basic layout
  - Minimal UI elements
  - 8 cards (4 pairs)
  - No distractions or extra features
  - Focus on core gameplay only

---

### 2. Complex Version
- File: `complex_hard_game.html`
- Features:
  - 16 cards (8 pairs)
  - Countdown timer (60 seconds)
  - Level display
  - Score + combo system
  - Hint system (limited uses)
  - Progress bar
  - Extra UI panels and instructions
  - More visual elements and distractions

---

## How the Study Works

1. Participants play the **simple version**
2. Participants complete a short survey
3. Participants play the **complex version**
4. Participants complete the same survey again

---

## Data Collected

You can measure:

### Performance Metrics
- Completion time
- Number of attempts
- Score achieved
- Success or failure (time limit)

### User Experience Metrics
- Ease of understanding (1–5 scale)
- Perceived difficulty
- Enjoyment level
- Cognitive load (how “overwhelmed” they felt)

---

## Research Question
How does simplistic vs. complex interface design impact end-user learning and interaction in a memory-based game?
